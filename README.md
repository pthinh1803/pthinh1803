## Project Style

I try to keep public repositories clean and useful for portfolio review:

- Clear README files with project goals, architecture, results, and limitations
- Public documentation separated from private reports or credentials
- Reproducible structure for source code, reports, images, and results
- Practical engineering notes instead of only screenshots
- Honest limitations and future improvement sections
